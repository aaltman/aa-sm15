: 1) Product change: Add cart "reservations"

Your interviewer asks you to add a new feature:

Your product manager comes to you with a problem and a proposal. Customers are trying to check out with 4-5 items in their cart, but by the time they get to the checkout page, some of the items are out of stock. This is causing a lot of frustration and lost sales. The product manager proposes that we add a "reservation" system to the cart. When a user adds an item to their cart, we will reserve the inventory for 60 minutes. If they don't check out within 60 minutes, the reservation will expire and the inventory will be released. If they do check out, the reservation will be released when the order is placed. The product manager wants to know if this is possible and how you would implement it.

Given these requirements, we need some way to "lock" inventory to a user when they add it to the cart. We also need a way to expire these locks after 1 hour or when the user checks out -- whichever comes first. We also need to make sure that our Availability service is aware of these locks and doesn't return them as available inventory. We have a couple options for how to implement this:

Great Solution: Use a new status flag
