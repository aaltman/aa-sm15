The last functional requirement we must address is the match creation / notification flow. Building off our previous design, we can add some business logic to the swipe service to check for a reciprocating swipe and generate a match if both users have swiped on each other. If a match is generated, we can store that match in a DB and fire a push notification to an external service, which eventually propagates the notification to the client.
