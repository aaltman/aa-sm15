The house robber problem (max non-adjacent sums) took a couple of attempts.

The successful one used iteration.  It had two base cases, a list of length 1 and a list of length 2.  length 1 is trivial, length 2 is the max of the values of the first two indices (since they can't sum) and then initializes the dp array in case we have length greater than 2.

We then iterate through to the right, and check backwards for matching dp values starting at i - 2 (not i - 1, since that's a neighbor).  We take the max of the dp values found and add it to the current, for the current dp, and check if we have a new max.  Then we return.

There was a gotcha here that iterating backwards over a list requires arguments to range as follows: range(i - 2, -1, -1).  The second argument -1 needs to be -1 rather than 0 because the range terminates BEFORE it gets to 0 or -1.  We want it inclusive of the zeroth element, so use -1.

The previous attempt tried to do recursive.  I think I didn't sufficiently think through my recursive cases in order to finish.  Reminder to self to draw out the tree and think hard about the base cases before I start writing code.

 

Problem statement:

 

You are given an integer array nums where nums[i] represents the amount of money the ith house has. The houses are arranged in a straight line, i.e. the ith house is the neighbor of the (i-1)th and (i+1)th house.

You are planning to rob money from the houses, but you cannot rob two adjacent houses because the security system will automatically alert the police if two adjacent houses were both broken into.

Return the maximum amount of money you can rob without alerting the police.
