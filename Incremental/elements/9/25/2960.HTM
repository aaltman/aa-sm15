: Putting it all together

Our full design might look like this:

We have two paths through our service: the write path and the query path. Via the write path we use an Ingestion Service to cut down the volume of events that we need to write to our backend ElasticSearch instance. On the query path we have 2 layers of caching, at the request level and in front of our elasticsearch instance. A re-ranking service grabs up-to-date like counts before returning to the user as what’s written to elasticsearch may be (intentionally) stale.
