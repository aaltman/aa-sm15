Replication Significance :

What is the significance of replication , In our system design we mostly have the requirement of making system durable , also thinking about whether the read throughput or write throughput in the given system is working as per our needs , and also with respect to different geographical location the write or read performance is not impacting the overall system .

  1. Durability
  2. Throughput
  3. Geographical location

Single Leader Replication :

Advantages :

  1. Read throughput is high : As we have mulitple followers from which we can read .
  2. Eventually consistent
  3. No write conflict as we only to one leader

Disadvantage :

  1. Low write throughput
  2. Single point of failure : If leader goes down , we have to rely on distributed consensus for the solution.

Multi-Leader Replication :

Advantages :

  1. High write throughput .
  2. No single point of failure as all leader is communicating to each other (all to all topology)
  3. If we have user in different geographical location , this set up is good .

Disadvantages :

  1. Write Conflict : Conflict resolution becomes crucial due to the possibility of conflicts arising from concurrent writes on different leaders.
    Timestamp in order to solve write conflict is also not a good choice
    Version vector can be used in order to deal with write conflicts.
    In Version vector : When we have concurrent writes , we can store both the writes (store siblings), Or we can choose the latest version write or CRDT(Conflict Resolution Data Type)

Leaderless Replication :

Instead of allowing user to write to one node at a time as we do in single or multi-leader , here a user can write to mulitple node at the same time . There is no concept of leader here .

Advantages :

  1. Relatively high write throughput. (lower than multi-leader but higher than single leader)
  2. Concept of Quorom (read/write) : Suppose you have 3 nodes 1 , 2 ,3 and you have N=3 and you have perform write on 2 nodes 1,2 and now you are reading from 2 other nodes 2, 3 . So we are guaranteed to see atleast one node which is having most up-to-date write.
    Also the concept of version vector is there , the write with latest version can be picked up .
  3. Read repair and Anti-entropy concepts are used here to solve write conflicts.
