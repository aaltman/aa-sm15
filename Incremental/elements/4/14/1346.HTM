: Projected radial search

[edit]

In the special case where the data is a dense 3D map of geometric points, the projection geometry of the sensing technique can be used to dramatically simplify the search problem.

The projected radial search approach requires that the 3D data is organized by a projection to a two-dimensional grid and assumes that the data is spatially smooth across neighboring grid cells with the exception of object boundaries.

Assumptions of a smoothly varying 2D grid in projected radial search are valid when dealing with 3D sensor data in applications such as surveying, robotics and stereo vision but may not hold for unorganized data in general.

In practice, the projected radial search technique has an average search time of O(1) or O(K) for the k-nearest neighbor problem when applied to real world stereo vision data.[4]
