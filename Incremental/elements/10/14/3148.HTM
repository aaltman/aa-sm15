1. In contests and sites like LeetCode, do the constraints look larger than what would allow the "obvious" DP? Then there is very likely a greedy observation somewhere in the problem

  2. How much freedom do we have to change the structure of the input data? For example, are we free to sort an input array because we just need to select "some" elements of the array rather than "in order"? In that case, assume that you are free to sort by whatever criteria you see fit and see if a greedy comes up

  3. Think about the first choice or modification that you can make. Does it make sense to take or modify the first/last/smallest/largest element, or can you show that it should be avoided?
